## Canonical execution

1. Use commit `ccfb15dbce720b324dd7a121f3eda2b23a49448b` or a reviewed descendant.
2. Run `.github/workflows/ch1-phase1-lability.yml`.
3. The workflow downloads the pinned, validated CHELSA analysis inputs from run
   `29380600763`, then executes script 77.
4. Confirm both `ch1-phase1-lability` and `Ch1 submission code CI` succeed.
5. Download `ch1-final-lability-results` from the successful workflow run.
6. Record the artifact digest, workflow run ID and commit SHA before manuscript
   tables or figures are copied.

spatial cohort. Rebuild it only when the observation cohort, CHELSA source or
climate-extraction code changes; do not silently mix climate tables and coefficient
files from different runs.

## Manuscript checks

Before submission, verify that:

- the text reports 102 complete taxa, not all 259 input taxa;
- the four pooled FDR-significant effects are described as small;
- the variation–responsiveness Spearman correlation is reported as -0.333;
- module summaries use medians and retain their sample sizes;
- hue magnitude is not given a scalar-angle p-value;
- `plasticity`, `adaptation`, `selection` and `evolutionary rate` are not claimed
  as demonstrated results;
- the old statement “zero FDR-significant within-species effects” has been removed
  from the active manuscript and README.

## Submission bundle

Create a durable bundle containing:

- Python package versions and any R `sessionInfo()` used elsewhere;
- Git commit SHA and workflow run ID;
- SHA-256 checksum manifest;
- a file-level README mapping outputs to manuscript tables and figures.

GitHub Actions artifacts are temporary working products. Deposit the frozen bundle
in a durable archive such as Zenodo when the manuscript is frozen.
